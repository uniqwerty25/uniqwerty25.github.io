Features of UniQwerty

•  No dead keys
•  No Alt Gr
•  No long press
1/6

This key acts as an accent only in the following cases.

• ` + vowel — only when a vowel follows immediately (no space)
• ~ + n — only when n follows immediately (no space)

In UniQwerty, only these two accent keys behave just as they look.
We once tried to move them closer — without Shift — but no key was free enough.
Luckily, they’re not used that often. Let’s call it a small mercy.
2/6

This key acts as an accent only in the following cases.

• q + a → æ
• q + o → œ
only when the vowel follows immediately (no space)

You probably don’t know any words that contain -qa- or -qo-.
So this combination safely serves only for accent input.
3/6

This key acts as an accent only in the following cases.

• , + vowel (a, e, i, o, u, y) — only when the vowel follows immediately (no space)
• , + c → ç — only when c follows immediately (no space)

You always insert a space after a comma in normal writing.
Therefore, there’s absolutely no chance of conflict when you type a vowel right after it.
4/6

This key acts as an accent only in the following cases.

• .. + vowel — only when the vowel follows immediately (no space)

When you type an ellipsis with two dots, you always add a space afterward.
So this pattern never interferes with your regular writing.
5/6

This key acts as an accent only in the following cases.

• ; + vowel — only when the vowel follows immediately (no space)

You always insert a space after a semicolon.
Therefore, it never conflicts with typing a circumflex accent.

(In some programming languages, the semicolon marks the end of a statement.
It’s a good habit to leave a space after it.)
